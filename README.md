# GL 분석적 절차·부정탐지 자동화

**🔗 라이브 데모 → https://jeongyeoju-gl-audit.vercel.app**

총계정원장(GL)을 **표본 없이 전수 분석**해, 감사인이 먼저 확인할 **이상 계정**과
**부정위험 전표**를 근거와 함께 좁혀주는 도구입니다.

- **분석적 절차** — 계정×월 기대구간을 만들고 로버스트 통계로 이탈을 선별 (감사기준서 520)
- **전표 레드플래그 스크리닝** — 11종 규칙을 전 전표에 적용해 확인 우선순위 산출 (감사기준서 240)
- **재무제표 수준 신호** — Beneish M-Score·Altman Z′를 참고 지표로 산출

> ⚠️ 이 저장소의 모든 데이터는 **가상제조(주) 합성 데이터**입니다.
> 실제 감사자료는 어떤 형태로도 포함되어 있지 않습니다.

---

## 핵심 설계 원칙

| 원칙 | 구현 |
|---|---|
| **판정에는 항상 근거** | 부정위험 점수 = 걸린 규칙 배점 합 ÷ 활성 규칙 만점 × 100. 화면의 "걸린 이유" 태그만으로 점수가 재현됩니다. |
| **회사마다 다른 원장 형식 흡수** | 열을 위치가 아니라 **이름(키워드)**으로 매칭하고, '차변/대변 헤더 행'을 앵커로 잡는 적응형 로더 |
| **자기참조 검증 금지** | 회계기간은 검증 대상인 거래일이 아니라 **독립 근거**(환경변수·설정 파일·파일 메타데이터)에서만 받습니다. 근거가 없으면 자동 통과시키지 않고 **분석 보류**로 처리합니다. |
| **실패 격리** | 한 단계가 실패해도 배치 전체가 멈추지 않습니다 |

## 빠른 시작

```bash
pip install -r requirements.txt

# 분석할 원장(.xlsx/.xls)을 data/ 에 넣고 실행
mkdir -p data && cp demo_data/가상제조_GL_2025.xlsx data/
python src/run_all.py
# → output/<회사명>/data, output/<회사명>/reports 에 결과 생성
```

### 회계기간 지정

날짜 범위 검증에는 거래일과 **독립된 근거**가 필요합니다. 우선순위는 다음과 같습니다.

1. `GL_FY_START` / `GL_FY_END` — 비달력 회계연도 지원
2. `GL_FY_YEAR` — 달력연도
3. `<BASE>/fiscal_period.json` — 설정 파일
4. `<BASE>/data/ledger_meta.json` — 로더가 파일명에서 추출 (자동 생성)

```bash
# 예: 4월~익년 3월 회계연도
GL_FY_START=2024-04-01 GL_FY_END=2025-03-31 python src/run_all.py
```

근거가 하나도 없으면 날짜 범위 검증을 **보류(analysis hold)** 하고 사유와 제안값을 리포트에 남깁니다.

## 파이프라인

| 단계 | 스크립트 | 하는 일 |
|---|---|---|
| 로드 | `adaptive_loader.py` | 형식이 다른 원장을 표준 스키마로 통일 |
| FR-02 | `fr02_validate.py` | 대차평형·계정 일관성·시산표 역산 대사 등 10종 무결성 검증 |
| FR-03 | `fr03_expectation.py` | 계정×월 기대구간 생성(전년 동월 기준) |
| FR-04 | `fr04_flagging.py` | 로버스트 z(중앙값·MAD)로 이상 후보 선별 |
| FR-05 | `fr05_anomaly.py` | Isolation Forest·벤포드 법칙 보조 탐지 |
| FR-09 | `fraud_screen.py` | 전표 레드플래그 11종 전수 스크리닝 |
| FR-10 | `fr10_classify.py` | 표준 계정 분류 |
| FR-11 | `fr11_fraud_scores.py` | Beneish M-Score·Altman Z′ |
| FR-14 | `fr14_export_json.py` | 웹 화면용 JSON 내보내기 |

`run_all.py`가 FR-02~FR-10을 순차 실행하며, 각 단계 실패는 격리됩니다.

## 결과 화면 (포트폴리오 사이트)

```bash
cd portfolio
npm install
npm run build          # 정적 사이트 생성 → out/
npx serve out          # 로컬 미리보기
```

Next.js 15 정적 내보내기(서버 없음). 대시보드는 **무결성 검증 → 이상 계정 → 계정별 상세 →
부정위험 전표 → 재무제표 신호** 순으로 읽도록 구성했고, `/report`는 인쇄용 보고서입니다.

`web/`은 같은 결과를 **단일 HTML 파일**로 담은 초기 버전입니다(서버·빌드 없이 열림).

## 테스트

```bash
python -m pytest tests/ -q      # 32 passed
```

회귀 테스트는 실제로 발견된 결함을 재현한 것들입니다.

- **날짜 처리** — Excel serial number(1899-12-30 기준) 오해석, 문자열·MM-DD 형식,
  기간 밖 오입력(전기/차기), 전기 비교자료 혼입, 비달력 회계연도, 회계기간 근거 부재
- **원장 적재** — 동일 구조 시트 다중 적재, 구조가 다른 시트 병합 방지,
  계정코드 없는 시트별 원장의 GL-TB 대사 정합
- **메타필드** — 결측이 `"nan"` 문자열로 바뀌어 부정위험 점수의 분모를 왜곡하던 문제
